## AWS Cloud9 이란?

> AWS Cloud9은 웹브라우저만으로 코드의 개발 및 실행 디버깅을 수행할수 있는 클라우드 기반 IDE(통합 개발 환경) 이다.

AWS Cloud9은 서버리스 애플리케이션을 개발할 수 있는 원활한 환경을 제공하므로 손쉽게 서버리스 애플리케이션의 리소스를 정의하고, 디버깅하고, 로컬 실행과 원격 실행 간의 전환을 할 수 있으며, 서버리스 개발에 필요한 모든 SDK,라이브러리 및 플러그인으로 개발 환경을 사전에 구성하여, 서버리스 개발을 보다 손쉽게 수행할수 있는 개발 환경을 제공한다.

AWS Cloud9의 협업 기능을 활용하여 클릭 몇번으로 개발 환경을 팀과 공유하고 프로그램을 함께 연결하여 코드에 대한 협업을 수행할 수 있다.
협업을 수행하는 동안 팀원들은 서로 입력하는 것을 실시간으로 보고 IDE 내에서 채팅도 할수있다.(오오 신기)

AWS Cloud9은 추가적인 비용은 없는데 구동하기 위해 필요한 EC2 인스턴스와 EBS에 대한 비용을 지불해야 한다.
또한 추가요금 없이 SSH를 통해 AWS Cloud9 개발 환경을 기존에 서비스 중인 Linux 서버에 연결할 수 있다.

AWS Cloud9 주요특징

> 1. 클라우드에서 사용하기 쉬운 클라우드 기반 IDE(Integrated Development Environment)
> 2. 내장형 GitHub와 내장형 Lambda BluePrint 지원
> 3. 내장 SAM Local을 활용하여 로컬 환경에서 테스트 , 디버깅 수행 가능
> 4. Lambda에 직접 배포하거나 GitHub에 업데이트 푸시 가능
> 5. AWS CodePipeline 구성을 통해 업데이트에 대한 빌드 자동화 가능

## AWS CodeBuild란?

AWS CodeBuild는 소프트웨어 개발에 필요한 소스코드를 컴파일하는 단계에서부터 테스트 후 소프트웨어 배포까지의 단계를 지원하는 완전 관리형 지속적 통합(Continuous Integrated) 서비스 이다.

CodeBuild 를 사용하면 자체 빌드 서버가 필요하지 않고 빌드 서버를 프로비저닝 하거나 운영/관리 및 확장을 수행할 필요가 없다.
또한 여러빌드를 동시에 처리할 수 있기 때문에 빌드를 위해 대기하지 않고 빠르고 효율적으로 빌드 작업을 수행할 수 있다.
AWS CodeBuild는 사전에 정의된 빌드환경을 생성해 CodeBuild에서 지원하는 사전 패키지 형태의 빌드도구와 런타임을 활용하여 자체 빌드 도구와 런타임을 AWS CodeBuild 에서 사용할 수 있다.
